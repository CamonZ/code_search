# import

Import a call graph JSON file into the database.

## Purpose

Load extracted Elixir/Erlang call graph data from a JSON file into the CozoDB database. This is the first step before running any queries. The JSON file should be generated by an Elixir call graph extractor tool.

## Usage

```bash
code_search --format toon import --file <JSON_FILE> [OPTIONS]
```

## Required Options

| Option | Description |
|--------|-------------|
| `-f, --file <FILE>` | Path to the call graph JSON file |

## Optional Flags

| Option | Description | Default |
|--------|-------------|---------|
| `-p, --project <NAME>` | Project namespace (allows multiple projects in same DB) | `default` |
| `--clear` | Clear existing data before import | false |

## Output

Returns import statistics: modules, functions, calls, structs, locations, specs, and types imported.

## When to Use

- Initial setup of a new codebase
- Refreshing data after code changes
- Loading multiple projects into a single database
